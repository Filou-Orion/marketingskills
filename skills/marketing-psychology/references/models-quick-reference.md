# Mental Models Quick Reference

One-line summaries and when-to-use guide for all mental models covered in the marketing-psychology skill.

---

## Lookup by Marketing Challenge

### "People aren't converting"
| Model | Apply How |
|-------|-----------|
| **Hick's Law** | Too many choices → reduce options |
| **Activation Energy** | Starting friction too high → simplify first step |
| **BJ Fogg** | Check: Motivation + Ability + Prompt all present? |
| **Paradox of Choice** | 7 options → 3 options |
| **Default Effect** | Pre-select the recommended option |
| **Status-Quo Bias** | Make switching feel safe and reversible |

### "People don't trust us enough"
| Model | Apply How |
|-------|-----------|
| **Authority Bias** | Add expert endorsements, certifications, "featured in" |
| **Social Proof / Bandwagon** | Show customer counts, logos, reviews |
| **Reciprocity** | Give value first (free content, free tools) |
| **Pratfall Effect** | Admit one weakness to increase credibility |
| **Mere Exposure** | Increase brand touchpoints across channels |
| **Liking / Similarity** | Use relatable founders, customer language |

### "People won't pay / price objections"
| Model | Apply How |
|-------|-----------|
| **Anchoring** | Show higher price first (competitor, enterprise tier) |
| **Framing** | "90% success" not "10% failure" |
| **Mental Accounting** | "$3/day" not "$90/month" |
| **Loss Aversion** | Frame as what they lose by not buying |
| **Decoy Effect** | Add inferior tier to make target tier obvious |
| **Charm Pricing** | $99 not $100 (for value products) |
| **Rounded Pricing** | $500 not $497 (for premium products) |
| **Rule of 100** | Under $100: use %. Over $100: use $ off. |

### "People sign up but don't activate"
| Model | Apply How |
|-------|-----------|
| **Goal-Gradient** | Show progress bars, completion percentage |
| **IKEA Effect** | Let users customize/build something |
| **Commitment & Consistency** | Get small commitments, then escalate |
| **Zeigarnik Effect** | "You're 80% done" creates pull to finish |
| **Endowment Effect** | Free trial creates ownership feeling |
| **Peak-End Rule** | Create a delightful moment in onboarding |

### "People leave / churn"
| Model | Apply How |
|-------|-----------|
| **Endowment Effect** | Show what they've built/accumulated |
| **Switching Costs** | Increase investment: integrations, data, workflows |
| **Status-Quo Bias** | Make staying the default, leaving require effort |
| **Loss Aversion** | Show what they'll lose ("Your 500 saved items will be deleted") |
| **Sunk Cost** | Remind of time/effort invested (use carefully) |

### "Growth has stalled"
| Model | Apply How |
|-------|-----------|
| **Theory of Constraints** | Find the one bottleneck. Fix only that. |
| **Local vs. Global Optima** | Are you optimizing the wrong thing? |
| **Pareto Principle** | Find the 20% driving 80% of results |
| **Compounding** | Consistent small gains beat sporadic big bets |
| **Flywheel Effect** | Build self-reinforcing loops |
| **Critical Mass** | Focus on one segment before expanding |

## All Models: One-Line Summaries

### Thinking Models
| Model | One-Line Summary |
|-------|-----------------|
| First Principles | Break to basic truths, build up from there |
| Jobs to Be Done | People hire products to get a job done |
| Circle of Competence | Stay where you have genuine advantage |
| Inversion | Ask "What would guarantee failure?" and avoid it |
| Occam's Razor | Simplest explanation is usually correct |
| Pareto (80/20) | 80% of results from 20% of efforts |
| Local vs. Global Optima | Don't optimize the wrong thing |
| Theory of Constraints | Find and fix the one bottleneck |
| Opportunity Cost | Every choice means giving up alternatives |
| Diminishing Returns | After a point, more investment yields less |
| Second-Order Thinking | Consider the effects of the effects |
| Map ≠ Territory | Your model isn't reality |
| Probabilistic Thinking | Think in odds, not certainties |
| Barbell Strategy | 80% safe bets + 20% wild bets |

|-------|-----------------|
| Fundamental Attribution Error | Blame process, not people |
| Mere Exposure | Familiarity breeds preference |
| Availability Heuristic | Vivid examples feel more likely |
| Confirmation Bias | People seek info that confirms existing beliefs |
| Lindy Effect | Old ideas outlast new ones |
| Mimetic Desire | People want what others want |
| Sunk Cost Fallacy | Past investment shouldn't justify future spend |
| Endowment Effect | People value what they already have more |
| IKEA Effect | People value what they helped build |
| Zero-Price Effect | Free is psychologically different from cheap |
| Hyperbolic Discounting | Immediate rewards beat future ones |
| Status-Quo Bias | People prefer the current state |
| Default Effect | Pre-selected options get chosen |
| Paradox of Choice | Too many options paralyze |
| Goal-Gradient | People speed up near the finish line |
| Peak-End Rule | Experiences judged by best moment and ending |
| Zeigarnik Effect | Unfinished tasks nag the mind |
| Pratfall Effect | Small flaws make the competent more likable |
| Curse of Knowledge | Experts forget what it's like not to know |
| Mental Accounting | Same money feels different in different contexts |
| Regret Aversion | People avoid potential regret |
| Bandwagon / Social Proof | People follow the crowd |

### Persuasion
| Model | One-Line Summary |
|-------|-----------------|
| Reciprocity | Give first, people feel obligated to return |
| Commitment & Consistency | Small commitments lead to larger ones |
| Authority Bias | People defer to experts |
| Liking / Similarity | People say yes to those like them |
| Unity Principle | Shared identity drives influence |
| Scarcity / Urgency | Limited availability increases value |
| Foot-in-the-Door | Small request → then larger request |
| Door-in-the-Face | Large request → then reasonable request |
| Loss Aversion | Losing hurts 2x more than gaining feels good |
| Anchoring | First number sets the reference |
| Decoy Effect | Inferior option makes target option obvious |
| Framing | Same facts, different frame = different perception |
| Contrast Effect | Things look different depending on what's next to them |

### Pricing
| Model | One-Line Summary |
|-------|-----------------|
| Charm Pricing | $99 feels much cheaper than $100 |
| Rounded Pricing | $500 feels premium; $497 feels value |
| Rule of 100 | Under $100: show %. Over $100: show $ |
| Good-Better-Best | Middle tier wins when framed by extremes |
| Mental Accounting (Pricing) | "$1/day" beats "$30/month" |
